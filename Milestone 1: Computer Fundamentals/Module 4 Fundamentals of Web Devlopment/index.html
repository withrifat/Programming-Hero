Web Development Notes

Building Blocks of Web Development

1. Frontend Development

  • What it is: The part of the website or web app that users interact with directly.
  • Technologies: HTML, CSS, JavaScript, frameworks/libraries like React, Angular, Vue.js.
  • Main tasks: Creating layouts, styling, making interactive elements, responsive design for different devices.
  • Skills: UI/UX understanding, JavaScript, CSS preprocessors (SASS, LESS), version control (Git).

2. Backend Development

  • What it is: Server-side development, dealing with databases, application logic, and server configuration.
  • Technologies: Languages like Python (Django, Flask), JavaScript (Node.js), Ruby (Rails), PHP, Java, databases (SQL, NoSQL).
  • Main tasks: Handling data storage, user authentication, API development, server logic.
  • Skills: Server management, database design, RESTful APIs, security best practices.

3. Full Stack Development

  • What it is: A developer who can work on both frontend and backend.
  • Technologies: Combination of frontend and backend tools.
  • Main tasks: Building complete applications, integrating frontend with backend, troubleshooting full system.
  • Skills: Versatile knowledge across both domains, problem-solving, project architecture.

Website Categories / Types

  • Personal Website: Portfolio, blogs, resumes, hobby sites.
  • Business/Agency Website: Company profile, service listings, client engagement.
  • E-commerce Website: Online stores with product listings, cart, payment systems (e.g., Amazon, Shopify).
  • Educational Website: Online courses, tutorials, learning management systems (LMS).
  • Entertainment Website: Streaming platforms, gaming sites, media portals.
  • Others: News portals, social networking sites, government websites, non-profits, forums.

How the Web Works: Domain Names and URLs

  • Domain Name: Human-readable address of a website (e.g., www.example.com).
  • IP Address: Numeric address that servers use (e.g., 192.0.2.1). DNS translates domain names to IP addresses.
  • URL (Uniform Resource Locator): Complete address to access a specific resource on the internet, includes protocol (http/https), domain, path, and parameters.
  • Web Hosting: Space on a server where website files are stored. When users visit a URL, the browser sends a request to the server to fetch files.
  • Protocols: HTTP and HTTPS are the main protocols for communication between browsers and servers. HTTPS adds encryption for security.
  • DNS System: Domain Name System resolves domain names to IP addresses to direct internet traffic correctly.
  • Web Browsers: Software (Chrome, Firefox, Safari) that fetches, renders, and displays websites.

4-6. How to Explore New Websites and Think About Design and Features

Exploring new websites effectively involves:

  • Analyze the User Interface: Observe layout, color schemes, typography, and overall user experience.
  • Evaluate Features: Identify interactive components, navigation ease, responsiveness, and any unique functionalities.
  • Consider Target Audience: Understand who the website is designed for and how it caters to their needs.
  • Look for Trends: Notice modern design trends like minimalism, dark mode, animations, or micro-interactions.
  • Take Notes: Document ideas, what you like or dislike, and potential improvements.
  • Use Tools: Tools like Chrome DevTools or extensions can help inspect website structure and performance.

4-7. How to Utilize AI Tools as a Developer?

  • Code Assistance: Use AI-powered code completion and suggestion tools like GitHub Copilot or ChatGPT to speed up coding.
  • Bug Detection: Leverage AI to detect bugs and suggest fixes automatically.
  • Testing Automation: Generate test cases or automate testing with AI-driven tools.
  • Documentation: Automatically generate or improve documentation using AI summarization.
  • Learning & Prototyping: Use AI to learn new frameworks, get examples, or quickly prototype ideas.
  • Chatbots & Assistants: Integrate AI chatbots into projects for better UX or internal tool automation.

How to Use VS Code Terminal, CMD, and PowerShell (Part One)

Understanding and using terminals effectively can boost productivity. Key points include:

  • VS Code Terminal: Integrated terminal inside VS Code allows you to run command-line commands without leaving the editor.
  • Opening Terminal: Use Ctrl + ` to toggle terminal in VS Code.
  • Running Commands: Run build commands, git commands, or npm scripts directly.
  • Basic CMD & PowerShell: Windows Command Prompt (CMD) and PowerShell are command-line interfaces for Windows OS.
  • Differences: PowerShell is more powerful and supports scripting with objects, while CMD is more basic and traditional.

4-9. How to Use Terminal (Part Two)

  • Navigation: Use commands like cd to change directories, ls or dir to list files.
  • File Operations: Create (touch or New-Item), delete (rm or del), and move files.
  • Running Scripts: Execute scripts or programs directly from terminal.
  • Using Git: Perform version control commands like git status, git commit, and git push.
  • Customizing Terminal: Adjust colors, fonts, and behaviors to improve comfort and efficiency.
